HTML

  • html lang=en - the page is written in english
  • <head> - contains the metadata for our web application
  • <title> - defines the title for the webpage
  • <style> - defines the CSS styles associated with a website
  • <base> - defines the base URL for a webpage
  • <link> - connects an exdternal resource to the HTML document
  • <meta> - defines such things like character set, descriptions, keywords, author, and viewport
  • <script> - embeds client-side scripts in the HTML document

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Structure - Selector (Declaration Block)
  • * is a wildcard symbol - to make all elements follow
  • .card - adding the dot before makes it so it applies to all 'cards'

Git

  • pwd - print working directory
  • ls - shows a list of working files in your directory
  • cd - change directory
  • 'cd ..' - go back one level
  • mkdir - make directory
  • touch - creates a file inside a directory (touch first-day.txt)
  • open - open the file (open first-day.txt)
  • Git must precede every statement
  • -b - creates a new branch (git checkout -b feature/starter-code)
  • add - adds modifications in the current working branch to the staging
  • git pull origin main - pull - to receive a branch's modifications into the local environment
  • origin - indicate the source of the pull will be in the GitHub repo
  • Main - indicates the branch

JavaScript

  • Variables - containers that store values
  • String - Sequence of text (within "quatation marks")
  • Number - just plain numbers, do not have quotes around them
  • Boolean - a true/false value
  • Array - Structure that allows you to store multiple values in a single reference
  • Object - can be anything
  • Comments - snippets of text that can be added along with code (starts with /* and ends with */)
  • Comments without line breaks can also be behind two slashes --> //comment
  • Index - location defined with a unique number
  • Index lists always start at 0 --> (0,1,2,3,4...)
  • Control Flow - from top to bottom, how the code is read
  • Loops - uses the predictable pattern of indexes to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Loops must know the shape's index
  • Loops must know the array's length
